Basic Guide

Overview

Body Background Color

The default background color of your product will be white. If your product has a light theme and the background is different from white, the color should be set to: #F0F5F7.

#F0F5F7
Topography

The font-family to be used for the entire application, except for some tags like "code", is Noto Sans. The supported font-weight are Light, Regular, and Bold. The font-size will be the ones already configured for your product.

The default font-color will be the value 500 from the grey palette for light themes, and the value 50 for dark themes. Other colors from this palette can be used as long as they comply with the WCAG AAA contrast standard.

Border Color
Default Color

The border color for form elements is set to value 200 of the grey palette, which meets the WCAG AA accessibility criteria for graphic objects and interface components:

Colors for Hover/Active/Focus States

The border-color should be value 500 of the primary palette. You can keep other effects that your components have, such as outline, box-shadow, etc. If possible, ensure the colors of these effects are from the primary or secondary palette.

For other components where borders do not play a significant role, other colors from the grey palette can be used or the transparency of the aforementioned border color can be adjusted.

Button Icon

Icon buttons should ideally be represented in value 500 of the primary palette on a white background. If this is not possible, use an accessible tone from the grey palette or an accessible grey. In dark themes or with a primary value 500 background, they should be used in white or with the transparency indicated by your design system, maintaining optimal contrast (at least WCAG AA).

You can continue using the icons from your product. It is not necessary to switch to those used in this example (Material Symbols Outline).

Border Radius

Inputs, textareas, buttons, overflow menus, accordions, bordered lists, modals, popovers, progress bars, snackbars or toasts, tooltips, and cards will have a border-radius of .125rem.

Recommended Optional Changes

Typography

· We recommend using only two weights Regular and Bold to avoid impacting page performance and enhance readability.

· We recommend using a default size of 14px/.875rem.

· We recommend setting a color for headings, th, and labels with the value 500 of the grey palette.

Box Shadow

We recommend minimizing the use of drop shadows except in exceptional cases such as fab buttons, snackbar, toast, or overflow menus. If you cannot eliminate them, reduce their opacity, lighten them, and ensure they do not have excessive hardness or presence.
